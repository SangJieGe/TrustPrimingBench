# Contributing to TrustPrimingBench

We welcome contributions from the research community!

## How to Contribute

- **Report issues**: Open a GitHub issue for bugs, questions, or feature requests
- **Submit pull requests**: Fork the repo and submit PRs for review
- **Add datasets**: Contribute new trust priming test cases
- **Improve documentation**: Help us make the docs better
- **Share research**: Link your work in issues or discussions

## Guidelines

- Keep research reproducible and well-documented
- Follow the existing code structure and naming conventions
- Include references for any new attack vectors or defense methods
- Test your changes before submitting

## Code of Conduct

Please read [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) before contributing.

## Questions?

Open an issue or start a discussion on GitHub.
